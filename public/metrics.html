QREP Metrics Reference

Explanation of how the metrics are calculated in QuantStats.

Performance Ratios

These ratios measure risk-adjusted returns - how much return you're getting for the risk you're taking.

Sharpe Ratio

What it is: Measures excess return per unit of total risk (volatility). Named after William Sharpe (1966).

Where it's useful:
  • Comparing investments with different risk levels
  • Evaluating portfolio managers
  • Most widely used risk-adjusted performance metric
How it's computed: 
Sharpe = (Portfolio Return - Risk-Free Rate) / Portfolio Volatility

Where:
- Portfolio Return = Annualized return (CAGR)
- Risk-Free Rate = Treasury rate (default: 0%)
- Volatility = Standard deviation of returns (annualized)
Interpretation:
  • Sharpe > 1.0 = Good
  • Sharpe > 2.0 = Very good
  • Sharpe > 3.0 = Excellent
  • Sharpe < 0 = You're losing money vs risk-free rate

Limitation: Treats all volatility equally (up and down movements), but investors typically only care about downside risk.

Sortino Ratio

What it is: Similar to Sharpe, but only penalizes downside volatility. Named after Frank Sortino.

Where it's useful:
  • When you care about downside risk specifically
  • Evaluating strategies that have asymmetric return distributions
  • Better for comparing strategies with different return patterns
How it's computed: 
Sortino = (Portfolio Return - Risk-Free Rate) / Downside Deviation

Where:
- Downside Deviation = Standard deviation of ONLY negative returns
Interpretation:
  • Higher is better (same scale as Sharpe, but typically higher values)
  • Sortino > Sharpe means positive skew (more upside volatility than downside)
  • Sortino < Sharpe means negative skew (more downside volatility)

Advantage over Sharpe: Doesn't penalize you for upside volatility (big gains).

Calmar Ratio

What it is: Measures return relative to maximum drawdown (worst peak-to-trough decline). Named after the California Managed Accounts Report.

Where it's useful:
  • Evaluating how much you risk losing to achieve returns
  • Important for strategies where drawdowns are psychologically difficult
  • Popular in hedge fund evaluation
How it's computed: 
Calmar = CAGR / |Max Drawdown|

Example:
- CAGR = 15%
- Max Drawdown = -30%
- Calmar = 15% / 30% = 0.50
Interpretation:
  • Calmar > 1.0 = Annual return exceeds max drawdown
  • Calmar > 3.0 = Excellent (rare)
  • Calmar < 0.5 = Poor risk/reward

Note: Very sensitive to the time period - one bad year can drastically change it.

Omega Ratio

What it is: Measures the probability-weighted ratio of gains vs losses relative to a threshold. Introduced by Keating and Shadwick (2002).

Where it's useful:
  • Captures entire return distribution (not just mean/variance)
  • Better for non-normal return distributions
  • More comprehensive than Sharpe for complex strategies
How it's computed: 
Omega = Sum of (Returns above threshold) / Sum of (Returns below threshold)

In QPulse (threshold = 0%):
- Numerator: Sum of all positive daily returns
- Denominator: Sum of absolute values of all negative daily returns

Example with 5 days:
Returns: +2%, -1%, +3%, -0.5%, +1%
Gains = 2 + 3 + 1 = 6%
Losses = |-1| + |-0.5| = 1.5%
Omega = 6 / 1.5 = 4.0
Interpretation:
  • Omega > 1.0 = Gains exceed losses
  • Omega = 1.0 = Break-even
  • Omega < 1.0 = Losses exceed gains

Note: The threshold is configurable in QuantStats (required_return parameter) but defaults to 0%.

Payoff Ratio

What it is: Measures average win size vs average loss size. Also called "Profit/Loss Ratio" or "Average Win/Loss Ratio".

Where it's useful:
  • Evaluating trading strategy quality
  • Understanding if wins are typically bigger than losses
  • Combined with Win Rate to assess strategy viability
How it's computed: 
Payoff Ratio = Average Win / |Average Loss|

Where:
- Win = Any day with return > 0% (strictly positive)
- Loss = Any day with return < 0% (strictly negative)
- Zero returns are excluded

Example:
- Winning days: +1.5%, +2.0%, +0.8% → Average Win = 1.43%
- Losing days: -0.5%, -1.2%, -0.8% → Average Loss = -0.83%
- Payoff Ratio = 1.43 / 0.83 = 1.72
Interpretation:
  • Payoff > 1.0 = Wins are bigger than losses on average
  • Payoff = 1.0 = Wins and losses are equal size
  • Payoff < 1.0 = Losses are bigger than wins
Relationship with Win Rate:

A strategy can be profitable with low Payoff if Win Rate is high, or with low Win Rate if Payoff is high.

Break-even: Payoff = (1 - Win Rate) / Win Rate

Profit Factor

What it is: Total gross profits divided by total gross losses. Simple measure of profitability.

Where it's useful:
  • Quick profitability assessment
  • Trading system evaluation
  • Easy to understand and communicate
How it's computed: 
Profit Factor = Sum of all Positive Returns / |Sum of all Negative Returns|

Example:
- Total gains: +10% across all winning days
- Total losses: -6% across all losing days
- Profit Factor = 10 / 6 = 1.67
Interpretation:
  • Profit Factor > 1.0 = Profitable (making money)
  • Profit Factor = 1.0 = Break-even
  • Profit Factor < 1.0 = Losing money
  • Profit Factor > 2.0 = Very good
  • Profit Factor > 3.0 = Excellent (rare in live trading)

Difference from Payoff Ratio: Profit Factor uses SUM of returns (total profit/loss), Payoff uses AVERAGE (mean win/loss size).

Returns & Performance

CAGR (Compound Annual Growth Rate)

What it is: The annualized rate of return assuming profits are reinvested.

How it's computed: 
CAGR = (Ending Value / Starting Value)^(1/years) - 1

Best Year

The highest annual return achieved in any single calendar year during the analysis period.

How it's calculated:
  • QuantStats groups daily returns by calendar year
  • Compounds the daily returns within each year to get annual return
  • Returns the maximum (best) annual return across all years
Example:

If analyzing 2020-2024 data with yearly returns:

  • 2020: +15%
  • 2021: +28%
  • 2022: -18%
  • 2023: +12%
  • 2024: +8%

Best Year = 28% (2021 had the highest annual return)

Source: quantstats.stats.best(returns, aggregate='A')

Worst Year

The lowest (most negative) annual return achieved in any single calendar year during the analysis period.

How it's calculated:
  • QuantStats groups daily returns by calendar year
  • Compounds the daily returns within each year to get annual return
  • Returns the minimum (worst) annual return across all years
Example:

Using the same 2020-2024 data:

  • 2020: +15%
  • 2021: +28%
  • 2022: -18%
  • 2023: +12%
  • 2024: +8%

Worst Year = -18% (2022 had the lowest annual return)

Source: quantstats.stats.worst(returns, aggregate='A')

Risk Metrics

Max Drawdown

What it is: The largest peak-to-trough decline in portfolio value.

How it's computed: 
Max Drawdown = (Trough Value - Peak Value) / Peak Value

Example:
- Portfolio reaches $100,000 (peak)
- Falls to $70,000 (trough)
- Max Drawdown = (70,000 - 100,000) / 100,000 = -30%

Volatility (Annualized)

What it is: Standard deviation of returns, annualized.

How it's computed: 
Volatility = Daily Std Dev * sqrt(252)

Where 252 = trading days per year
